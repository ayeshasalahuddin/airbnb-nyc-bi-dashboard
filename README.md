# Airbnb NYC Supply-Demand BI Dashboard

A complete Business Intelligence project analyzing New York City Airbnb Open Data
to help operations managers identify supply-demand gaps in real time.

## 🎯 Business Problem
A Regional Operations Manager needs a way to identify and predict geographic
supply-demand gaps in real time, since lagging historical data makes it difficult
to recruit hosts fast enough to meet demand surges.

## 📊 Dataset
- **Source:** [NYC Airbnb Open Data — Kaggle](https://www.kaggle.com/datasets/arianazmoudeh/airbnbopendata)
- **Size:** 102,599 rows × 26 columns (101,857 rows × 24 columns after cleaning)

## 🛠️ Tools Used
- **Python:** pandas, matplotlib, seaborn, scipy
- **BI Tool:** Microsoft Power BI Desktop
- **Methodology:** Design Thinking Framework (Empathize, Define, Ideate, Prototype, Test)

## 🧭 Design Thinking Process
This project followed the 5-stage Design Thinking framework:
1. **Empathize** — Interviewed Mohammad Shahmeer (Asst. Manager, Livedin) + empathy map
2. **Define** — Formal problem statement and How Might We statements
3. **Ideate** — 20+ sticky note ideas generated by both team members
4. **Prototype** — Hand-drawn dashboard wireframe
5. **Test** — AI domain expert feedback and team reflection

## 🔑 Key Findings
- Manhattan & Brooklyn are critically undersupplied despite having the most listings
- 7,000 listings flagged as at-risk; 4,000 completely inactive
- Near-zero correlation between price and review rating
- Staten Island holds the top 10 most expensive neighbourhoods — an unexpected finding
- Booking activity crashed in 2020 (COVID-19) and recovered by 2022
